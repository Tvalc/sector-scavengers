# VIRAL VIDEO BRIEFS

## Sector Scavengers
### Platform-Specific Editor Briefs + Developer Capture Brief

## Important note

No video can be honestly guaranteed to go viral.

What **can** be done is to dramatically increase the odds by:

- making the concept immediately legible
- leading with a painfully clear hook
- cutting platform-native versions instead of one generic trailer
- making sure the captured footage supports the story being told

This document is designed to maximize the odds of breakout performance on:

- TikTok
- X
- Reddit
- YouTube

using separate edits built from the same core material.

---

## 1. Core Marketing Truth

The game should not be sold as:

- “a cool sci-fi card game”
- “a systems-heavy extraction roguelite”
- “a stylish salvage sim”

Those are true, but too broad.

It should be sold as:

> **A debt-driven sci-fi roguelite where every crew member you save drags you deeper into the machine.**

That is the hook.

That is the meme.

That is the discussion prompt.

That is the emotional differentiator.

The best-performing videos will repeatedly orbit these tensions:

- “I woke up with debt”
- “every recruit costs me more debt”
- “I need to scavenge to survive”
- “if I expand, am I becoming the corporation?”
- “the systems are exploiting me, but I’m also learning to exploit them back”

---

## 2. Content Strategy Overview

Do **not** make one trailer and crop it for all platforms.

Instead make:

### A. TikTok
- emotion-first
- high-speed
- subtitle-driven
- big conceptual hook in first 1-2 seconds

### B. X
- system-hook first
- highly loopable
- dense with “cool” and “clever”
- optimized for reposting by game devs / indie fans / systems-game players

### C. Reddit
- authenticity-first
- devlog / “look at this mechanic” framing
- less ad-like
- more explanatory and discussion-friendly

### D. YouTube
- strongest overall pitch
- can sustain more structure and payoff
- should include both:
  - a short-form version
  - a fuller trailer/devlog version

## 4. TikTok Brief

## Goal

Stop the scroll immediately and make viewers understand:

> “Oh, this is the debt sci-fi game where saving people makes your life worse.”

## Runtime

- **20 to 35 seconds**

## Format

- **9:16 vertical**
- large subtitles always on screen
- aggressive pacing
- 1 idea per line

## Primary concept

**A worker wakes in debt, dives derelicts for survival, and every person they save deepens the debt trap.**

## Opening hook options

Use one of these in the first 1.5 seconds:

1. **“I’m making a roguelite where every crew member you save puts you deeper in debt.”**
2. **“You wake up owing the company six figures... and every recruit costs more.”**
3. **“This is a sci-fi roguelite where debt is the real enemy.”**
4. **“I made a game where helping people is financially catastrophic.”**

## Suggested structure

### Version A — strongest conceptual hook
0:00 - 0:02  
Hook text + strongest visual of debt panel / wake screen / dramatic line

0:02 - 0:06  
Show cryo wake UI + debt consequence

0:06 - 0:12  
Show dive loop:
- SCAVENGE
- REPAIR
- EXTRACT
- collapse risk

0:12 - 0:18  
Show reward / results / debt serviced / consequence

0:18 - 0:24  
Show sector / station / crew / hub growth

0:24 - 0:30  
End on:
- “At what point are you no longer surviving the corporation… and becoming it?”

## Visual priorities

- debt panel
- cryo wake button
- mission control
- card choice moment
- collapse risk
- results
- hub / station board

## Editing style

- very fast cuts
- hard subtitle punch-ins
- 1 subtle sound motif repeated
- no dead air
- no long UI browsing

## Text style

- bold
- high contrast
- center/lower-third
- each line readable in under 1 second

## Audio direction

- tense, pulsing sci-fi rhythm
- one strong stinger on “debt”
- one stronger stinger on “wake”
- one hit on “collapse”

## CTA

Do not end with “wishlist now” in the first version.

End with a conversation prompt:

- “Would you wake more crew or pay down the debt?”
- “Is this survival… or management?”
- “Would you extract now or risk one more dive?”

TikTok comments matter more than a polished CTA.

## Suggested structure

0:08 - 0:15  
Dive loop with collapse pressure

0:15 - 0:22  
Results + debt servicing

0:22 - 0:28  
Station / sector implication

- less subtitle-heavy
- more “look at this system” than “performative reveal”

## X post copy suggestion

- “Would love feedback on whether this macro loop reads clearly.”
- “Would you pay down debt or expand?”

## 6. Reddit Brief

## Goal

Make it feel like a real devlog / mechanic reveal, not an ad.

## Runtime

- **30 to 60 seconds**

## Format

- 16:9 or square
- subtitles helpful but not mandatory

## Primary concept

Show a **problem + solution + payoff**.

Reddit responds better to:

- “I realized my game didn’t feel real, so I changed X”
- “I’m trying to make debt the core loop in my sci-fi roguelite”
- “Would this progression hook work for you?”

## Best Reddit framing options

1. **“I’m turning my sci-fi salvage prototype into a debt-driven roguelite. Does this loop read clearly?”**
2. **“Trying to solve a design problem: how do I make saving crew feel costly instead of purely beneficial?”**
3. **“I rebuilt my progression loop so every recruit adds debt. Here’s the current version.”**

## Suggested structure

0:00 - 0:08  
State the design problem

0:08 - 0:20  
Show the relevant UI/gameplay

0:20 - 0:35  
Show the intended loop:
- wake
- debt rises
- dive
- extract
- debt service

0:35 - 0:50  
Show where the system gets interesting:
- expansion vs debt
- sectors
- named recruits

0:50 - 0:60  
Ask one specific discussion question

## Editing style

- slower than TikTok
- more informational
- less hype, more clarity
- more comfortable with showing raw gameplay UI

## CTA

One focused question only, for example:

- “Does this make helping crew feel meaningfully costly?”
- “Would you understand this loop from the UI alone?”
- “What would make the debt pressure clearer?”

## 8. Editor Brief - General Direction

Give this to the video editor regardless of platform cut.

## Tone

- oppressive but seductive
- beautiful systems hiding cruel incentives
- premium indie, not parody prototype

## Key emotional beats

1. **Debt shock**
2. **Risk temptation**
3. **Small victory**
4. **Moral unease**
5. **Expansion dread**

## What the editor must understand

This is not a “look how many systems I have” trailer.

It is a trailer about:

- obligation
- extraction
- escalation
- complicity

## Keywords for edit tone

- neon
- scarcity
- pressure
- salvage
- debt
- consequence
- frontier
- corporate dread

## Visual motifs to repeat

- debt number / bar
- cryo wake
- card choice
- collapse risk
- station board
- signal ticker

## Things to avoid

- long static menu browsing

## 9. Developer Capture Brief

The editor can only cut a great video if the developer captures the right material.

This is the capture plan.

## Capture principles

Every clip should answer at least one of these:

- What is happening?
- Why does it matter?
- What changed?
- Why should the viewer care?

If a capture does not communicate one of those, it is probably B-roll only.

---

## A. Must-Capture Gameplay Clips

### Hub / Station
- clean idle hub wide shot
- debt panel visible
- cryo button highlighted
- mission control highlighted
- inventory open
- signal ticker active
- station board with multiple derelicts

### Cryo
- opening cryo panel
- attempt to wake with not enough resources
- successful wake
- named recruit wake if available
- assignment flow

### Missions
- mission control open
- blocked mission state (“need crew”)
- successful mission start
- mission completion / collection

### Dive
- entering a dive
- multiple rounds of:
  - SCAVENGE
  - REPAIR
  - EXTRACT
- collapse or near-collapse moment
- discovery / loot moment
- visible run result moment

### Results
- successful extract
- collapse result
- debt serviced text
- sector display
- any unlock messaging

### Progression
- station growth
- sector unlock
- recruit arrival
- debt warning / threshold

---

## B. Capture Variants

For each important sequence, capture:

1. **clean version**
   - no cursor if possible
   - no debug noise
   - no misclicks

2. **alt version**
   - slightly different timing
   - different route

3. **failure version**
   - especially for collapse, blocked wake, blocked mission, debt warning

This gives the editor options.

## D. Capture quality requirements

- capture at highest stable resolution available
- capture at stable frame rate
- avoid showing obviously broken UI if the clip is meant to sell the game
- if UI is still rough, use framing text to guide attention
- capture enough dead space around moments for editor trimming
